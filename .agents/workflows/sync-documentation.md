---
description: How to automatically sync documentation with recent code changes
---

1. Analyze the codebase for changes in API, Types, or DB schemas.
2. Run the internal documentation sync tool or script (if available).
3. Update `.agents/rules.md` to reflect any architectural shifts.
4. Update `README.md` to ensure setup and feature lists are accurate.
5. Create or update `.agents/schema-snapshot.md` if DB changes occurred.
6. Verify all links in the documentation are correct and relative.
